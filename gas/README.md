# GASバックエンド セットアップ手順

Radio Meeting Scheduler の「共有調整ページ」（調整さん相当機能）を動かすための
Google Apps Script（GAS）バックエンドのデプロイ手順です。**所要時間は約5分、無料です。**

## 手順

### 1. GASプロジェクトを作成

1. https://script.google.com/ を開く（Googleアカウントでログイン）
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を「Radio Meeting Scheduler API」などに変更

### 2. コードを貼り付け

1. エディタに最初からある `function myFunction() {...}` を全部削除
2. この隣にある [Code.gs](Code.gs) の中身を全部コピーして貼り付け
3. 保存（Ctrl+S）

### 3. Webアプリとしてデプロイ

1. 右上の「デプロイ」→「新しいデプロイ」
2. 歯車アイコン →「ウェブアプリ」を選択
3. 設定：
   - **説明**: 任意（例: v1）
   - **次のユーザーとして実行**: 「自分」
   - **アクセスできるユーザー**: **「全員」** ← 重要！ゲストさんが回答するために必要
4. 「デプロイ」をクリック
5. 初回は「アクセスを承認」→ 自分のアカウントを選択 →
   「詳細」→「（プロジェクト名）に移動」→「許可」
6. 表示される **ウェブアプリのURL**（`https://script.google.com/macros/s/xxxx/exec`）をコピー

### 4. フロントエンドにURLを設定

コピーしたURLを `src/config.js` の `API_URL` に貼り付けます：

```js
export const API_URL = "https://script.google.com/macros/s/xxxx/exec";
```

その後、コミット＆プッシュすると GitHub Pages に反映されます。

## データの保存場所

初回アクセス時に「Radio Meeting Scheduler データ」というスプレッドシートが
自分のGoogleドライブに自動作成され、そこにイベントと回答が保存されます。
スプレッドシートを直接開いて回答を確認・編集することもできます。

## コードを更新した場合

Code.gs を書き換えたら、「デプロイ」→「デプロイを管理」→ 鉛筆アイコン →
バージョンを「新バージョン」にして「デプロイ」。**URLは変わりません。**
（「新しいデプロイ」にするとURLが変わってしまうので注意）
